# Table Catalog — Level 0 & Level 1 (Xero Data)

_Companion to `ACCOUNTANTS_GUIDE_XERO_DATA.md` — read that one first for the conventions and caveats. This document just lists every table, in both levels, with a plain-English description of what it holds. Useful as a reference while you're working through an audit and need to find (or ask IT for) a specific piece of data._

**One important shortcut: every table below exists, with the same name and the same meaning, in both Level 0 and Level 1.** The two levels hold the *same rows* — Level 0 uses Xero's own field names, Level 1 uses standardized/renamed field names — but the tables themselves line up one-to-one. So this catalog only needs to describe each table once. (The one exception is the three **reconciliation tables** at the very end, which only exist in Level 1 — see that section.)

## Reference data (one row per "thing," not per transaction)

| Table | What it holds | Rows / companies |
|---|---|---|
| `dim_organisation` | The 7 companies themselves — name, country, base currency, fiscal year end date, status. Your starting point for "which company is this." | 7 rows (1 per company) |
| `dim_account` | The full chart of accounts across all 7 companies — account code, name, type (Asset/Liability/Equity/Revenue/Expense), and Xero's own reporting classification. | 874 accounts, all 7 companies |
| `dim_contact` | Every customer and supplier, in one combined list. A flag on each row shows whether it's used as a customer, a supplier, or both — see the caveat in the main guide about this not being split into two clean lists yet. | 1,798 contacts, all 7 companies |
| `dim_currency` | The currencies each company has enabled/uses. | 30 rows, all 7 companies |
| `dim_tax_rate` | Tax rates/types set up in each company (e.g. GST, VAT, zero-rated). | 118 rows, all 7 companies |
| `dim_item` | Products, services, and stock items that can be bought or sold — from Xero's "Items" feature (not every company uses this). | 2,127 items, 6 of 7 companies |
| `dim_tracking_category` | "Departments" — custom reporting splits a company has set up in Xero (e.g. by location, project, or division). Not every company uses this feature. | 8 categories, 6 of 7 companies |
| `dim_tracking_option` | The specific values within each tracking category above (e.g. if the category is "Location," the options might be "Sydney," "Auckland," etc.). | 62 rows, 6 of 7 companies |
| `dim_branding_theme` | Invoice/document template designs set up in Xero. Administrative only — not financial. | 15 rows, all 7 companies |
| `dim_user` | Xero user accounts (who has login access to each company). Administrative only — not financial. | 151 rows, all 7 companies |

## Transactions

|---|---|---|---|

## Reconciliation checks — **Level 1 only**

These three tables don't exist in Level 0. They're not source data — they're the output of comparisons we've already run between our own numbers and Xero's, specifically to help with exactly this kind of audit review. If you're checking whether the numbers "make sense," these are the most direct place to look.

| Table | What it compares | What we found |
|---|---|---|
| `recon_trial_balance` | Our general ledger totals vs. Xero's own Trial Balance report, account by account. | Income statement accounts: 98% match exactly. Balance sheet accounts: only ~15% match — expected, see the conversion-balance note in the main guide. 429 rows (one per company + account combination). |
| `recon_profit_and_loss` | Our general ledger totals vs. Xero's own Profit & Loss report for the current period. | 94% match exactly; the small remainder is immaterial amounts Xero's own report doesn't even display. 85 rows. |
| `recon_document_facts` | Invoice/bill/payment/credit-note/etc. totals vs. the general ledger directly, account by account, across all history. | Matches well for most companies and transaction types. Two companies (Aqua Pharma Inc, Aquatiq Ltd) show a real, unresolved gap — see the main guide for detail. 1,453 rows. |

Each row in these three tables shows: the company, the account, our number, Xero's (or the ledger's) number, and the difference — so you can see exactly where and by how much something doesn't tie, rather than just a pass/fail.
